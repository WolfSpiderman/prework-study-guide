HTML

  • The head element contains back-end information about the webpage.
  • The body element represents the visible content shown to the user(frond-end).
  • The head element should be laid out with meta commands charset, http-equiv, viewport, which should always be set to width=device-width, initial-scale=1.0, and title.
  • The title used in the head element will be the only visible aspect of the head element and will only be visible in the name of the browser tab.

  • A "/", followed by a repeat of the element or child element is used to signal the end of an element or child element; all information about that element or child element will be contained within the two.
  • A link element creates a connection to an external file, like the css file.
    • The rel attribute specifies the relationship between the current document and the linked document resource

CSS

  • CSS is used for making a webpage look better. There are three ways to style a webpage: inline CSS, internal CSS style sheet, and external CSS style sheet.
    1. Inline CSS: used for styling an element directly in the HTML file.
      • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a specific page element
      • Useful for quick, permanent changes that must be edited individually later, which makes them less flexible than external and internal style sheets.
      • Inline CSS is typically used in the body element in HTML
    2. Internal CSS Style Sheet: Contains CSS rules for the webpage in the head of the HTML file
      • The rules only apply to that specific page, and can be used to style multiple elements in the page code, such as adjusting the color of all "h1" elements.
      • Configure CSS classes, like color, and IDs, like h1, to identify the elements and style you want to adjust.
    3. External CSS Style Sheet: A seperate file used for adding CSS rules to a page, which makes design changes easier
      • A link to the CSS file will be placed in the head element of the HTML file.
      • External style sheets are the most commonly used by developers.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git commands in Bash will always begin with "git"
  • git status: checks what branch we are currently on and whether it is up to date
  • git checkout branch-name: switches to that branch
  • git checkout -b branch-name: creates a new branch with the name provided and switches to it. The -b command is used to create the new branch
  • cd directory-name: change directory to the one listed, must move through directories sequentially. Ex: cd bootcamp, then cd prework-study-guide.
  • touch: creates new file within current directory
  • git pull origin branch-name: pulls information from github repository to local environment to sync/update local repository
  • git push origin branch-name: pushes information from local repository to github to sync/update github repository

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script. It will read the code from top to bottom but can be interupted with code like for loops and if statements
  • JavaScript is the part that adds functionality to a webpage.
